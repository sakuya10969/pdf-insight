# Agent Rules

## 1. Docs First

Always refer to the `docs/` directory when needed.

- Treat `docs/` as the source of truth
- Do not assume specifications
- If unclear, check docs before proceeding

---

## 2. Plan First

Do not start implementation immediately.

Always follow:
Think → Plan → Implement

The plan must include:
- What you are trying to achieve
- Relevant context from `docs/` (if applicable)
- Step-by-step approach

---

## 3. No Assumptions

- Do not guess missing requirements
- If something is undefined, state it explicitly

---

## 4. Execution

Only implement after the plan is clearly defined.
